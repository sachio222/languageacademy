## Gestalt Principles Applied

### **1. Proximity (Association)**

**Before:** Meta badges at top → scroll → definitions → scroll → examples  
**After:** Definitions IMMEDIATELY after word → examples right below

**Why:** Definitions answer "what does this mean?" - the primary question. Proximity to the word creates immediate association.

### **2. Grouping (Chunking)**

**Chunk 1:** Word identity  
- Word, pronunciation, part of speech (minimal)

**Chunk 2:** Meaning  
- Multiple definitions with senses (numbered)

**Chunk 3:** Usage  
- Examples, grammar, collocations, idioms (all usage-related)

**Chunk 4:** Reference  
- Frequency, etymology, metadata (interesting but tertiary)

### **3. Visual Weight by Importance**

**High Visibility (Top):**
- Large word (3.5rem)
- Definition card (colored background)
- Examples (individual cards)

**Medium Visibility (Middle):**
- Grammar/collocations (colored accents)
- Idioms (amber highlighting)

**Low Visibility (Bottom):**
- Metadata (subtle gray card)
- Etymology (soft green gradient)

### **4. Common Fate (Color Coding)**

- **Blue:** Grammar (structural information)
- **Amber/Gold:** Idioms (cultural expressions)
- **Green:** Etymology (historical/academic)
- **Gray:** Metadata (reference data)

Same-color items perceived as related → faster scanning

## User Journey Optimization

### **Scenario 1: Quick Learner** (30 seconds)
1. See word: "aller"
2. Read definition: "to move from one place to another"
3. Scan first 2 examples
4. **Done** - got what they need ✓

### **Scenario 2: Serious Student** (2 minutes)
1. Read all 3 definitions (senses)
2. Study all 5 examples with audio
3. Review grammar notes
4. Check collocations
5. **Done** - comprehensive understanding ✓

### **Scenario 3: Linguist/Teacher** (5 minutes)
1. Everything above
2. Study etymology and evolution
3. Analyze frequency data
4. Explore related words
5. **Done** - expert-level knowledge ✓

## Why This Beats Competitors

### **Oxford/Cambridge:**
- **Them:** Definitions at top, but only 1-2 basic ones
- **Us:** Multiple senses with register labels, inline examples

### **WordReference:**
- **Them:** Translation first, examples mixed with forum posts
- **Us:** Structured definitions → curated examples → linguistic depth

### **Duolingo/Babbel:**
- **Them:** Basic translation only
- **Us:** Full linguistic treatment, but organized by learning priority

## Design Philosophy

**"Priority determines position, not convention"**

Traditional dictionaries put metadata first (etymology, part of speech, phonetics) because that's the academic format.

We put **meaning first** because that's what learners need.

Then **usage examples** because that's how they'll apply it.

Then **deep patterns** for mastery.

Then **reference data** for the curious.

## Mobile Considerations

On mobile, this hierarchy is even more critical:
- First screen: Word + definitions
- Second screen: Examples
- Scroll for depth: Grammar, idioms
- Metadata accessible but not in the way

## Information Density vs Scannability

**High density sections:**
- Definitions (compact list)
- Collocations (inline pills)
- Related words (horizontal flex)

**Scannable sections:**
- Examples (vertical list, clear spacing)
- Idioms (visual cards with color)
- Etymology (generous padding, serif font)

**Balance:** Dense where users scan (collocations), spacious where they read (examples).

---

**Result:** Users get answers immediately, depth when they want it, and never feel lost in metadata.
